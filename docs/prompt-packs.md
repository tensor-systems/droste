# Prompt packs

Prompt packs are versioned, deterministic harness configuration. Exactly one
complete pack is resolved when a run starts; it supplies the system, user,
refinement, repair, and terminal-extraction templates, policy defaults, tips,
and provenance. Pack content can change by revision. The slot contract cannot.

Prompt packs are not RLM skills. Packs govern required machinery and are never
discovered or mutated mid-run. Additive, broker-loaded skills remain tracked by
#3 and are outside this contract.

## Stable slots

Every template is rendered with the same immutable value containing five string
slots. A template may use its phase's required subset and any other declared
slot, but may not invent slots, use attribute/index traversal, conversions, or
format specifications.

| Slot | Meaning |
|---|---|
| `capabilities` | Trusted environment/provider description generated by the host, plus explicit caller additions. |
| `budget` | The authorized iteration, subcall, depth, and output limits for this run. |
| `question` | The caller's question, unchanged. |
| `history` | Phase-specific facts: conversation context, prior answer/output, an error record, or the bounded extraction trajectory. |
| `output_contract` | The required output form for the current phase: fenced Python or terminal answer text. |

Slot values are inserted once. Braces inside a question, capability description,
or history are ordinary data and are not interpreted as nested templates.

## Artifact format

Packs are TOML so loading needs no runtime dependency. `schema_version`, `id`,
`revision`, `profile`, every template, a positive provenance statement, the
unable sentinel, tips, and typed policy defaults are required. Unknown fields
fail closed to catch misspellings. A benchmark and score are optional, but must
appear together.

```toml
schema_version = 1
id = "my-product.analysis.full"
revision = "2026-07-14.1"
profile = "full"
unable_sentinel = "unable to determine from the work so far"
tips = []

[provenance]
source = "my-product"
notes = "Prompt revision selected for the local analysis harness."
benchmark = "private-eval-v3"
score = "accuracy=0.84"

[policy_defaults]
enforce_contract = true

[templates]
system = """{output_contract}
{capabilities}
{budget}"""
user = """Question: {question}
{history}"""
refinement = """{history}
Continue. {output_contract}"""
missing_code_repair = """Return executable Python. {output_contract}"""
error_repair = """{history}
Repair the attempt. {output_contract}"""
extract_system = """Use only supported evidence. {output_contract}"""
extract_user = """Question: {question}
{history}"""
```

Literal braces authored in a template use normal Python-format escaping (`{{`
and `}}`). Braces in slot values need no escaping.

## Loading and resolution

Filesystem and package-resource reads are loader-boundary I/O. They return
frozen `PromptPack` values. Parsing, validation, catalog selection, and rendering
are pure functions over those values.

```python
from droste import RLMConfig, load_prompt_pack, run_rlm

pack = load_prompt_pack("./prompts/analysis.toml")
result = run_rlm(
    question,
    environment=environment,
    root_llm=root,
    subcalls=subcalls,
    config=RLMConfig(root_model="gpt-5", prompt_profile="full"),
    prompt_pack=pack,
)
print(result.prompt_pack.as_dict())
```

`resolve_prompt_pack` takes `(model, profile)` and selects, in order:

1. the complete caller pack;
2. a consumer catalog match for model family, then its generic match;
3. an engine catalog match for model family;
4. the engine's generic profile;
5. generic `full` only as compatibility fallback for an unknown profile.

Catalog selectors and packs are tuples/frozen values. Duplicate selectors fail
at construction. Selection never merges partial packs. A consumer catalog can be
passed to `run_rlm(..., consumer_prompt_catalog=catalog)`; model-family inference
is a pure selector and has no effect outside prompt resolution.

The installed engine includes `full`, `minimal`, and `none` generic packs. The
historical `tips_profile` selects the same profiles; `prompt_profile` is the new
spelling and wins when both are present. `enforce_contract=None` delegates that
single policy default to the pack, while explicit `True` or `False` wins.

## Compatibility and provenance

The built-in full pack preserves the existing REPL, structured-subcall,
refinement, repair, and bounded-extraction behavior. Existing complete
`system_prompt` overrides and the older user/refinement template arguments still
work; new integrations should provide a complete caller pack instead of creating
partial combinations.

Every `RLMResult` carries a frozen `PromptPackRecord` with pack ID, revision,
profile, resolution tier, model family, and benchmark provenance. The built-in
runner and CLI JSON expose that record as `prompt_pack`. It is an additive result
field, not a new trace protocol.
